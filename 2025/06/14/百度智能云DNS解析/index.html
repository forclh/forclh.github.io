背景：为了使用 Github Pages 中自定义域名的配置，需要到域名购买服务商进行域名解析配置。

一、什么是域名解析？

域名解析（DNS 解析）是将域名（如 www.example.com）转换为对应的IP 地址（如 192.168.1.10）的过程。由于 IP 地址难以记忆，域名解析通过 DNS 服务器完成这一转换，使得用户可以通过简单的域名访问网站或服务。

二、域名解析中关键字段的作用

在百度智能云（或其他 DNS 服务商）的域名解析设置中，通常需要填写以下字段：

1. 主机记录（Host Record）

  • 定义：主机记录是域名的前缀，用于指定解析的具体子域名或主域名。
  • 常见类型：
    • www：解析 www.example.com。
    • @：解析主域名（如 example.com）。
    • *：通配符，解析所有子域名（如 *.example.com）。

2. 记录类型（Record Type）

记录类型决定了解析的具体方式和用途。以下是常见的类型及其作用：

记录类型 作用 适用场景
A 记录 将域名指向一个 IPv4 地址 网站托管、服务器直连
AAAA 记录 将域名指向一个 IPv6 地址 IPv6 网络环境下的服务
CNAME 记录 将域名指向另一个 域名（别名） 使用 CDN、负载均衡或共享 IP 时
MX 记录 指定接收邮件的服务器地址 邮箱服务配置
TXT 记录 存储任意文本信息（如验证域名所有权） SPF 反垃圾邮件、SSL 证书验证
NS 记录 指定域名的 DNS 服务器 域名托管到其他 DNS 服务商时

3. 记录值（Value/Target）

  • 定义：记录值是解析的目标地址或域名，具体取决于记录类型。
  • 示例：
    • A 记录的记录值是一个 IPv4 地址（如 192.168.1.10）。
    • CNAME 记录的记录值是一个域名（如 cdn.example.com）。
    • MX 记录的记录值是邮件服务器域名（如 mail.example.com）。

4. TTL（Time to Live）

  • 定义：TTL 是 DNS 记录的缓存时间。指地方 DNS 缓存域名记录信息的时间，缓存失效后会再次到权威 DNS 获取记录值。
  • 建议：正常情况下使用 10分钟。
